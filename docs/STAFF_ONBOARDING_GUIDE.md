# Staff Onboarding Guide

Route: `/provider/settings/staff-onboarding`

## Tracked steps

- Invite accepted
- Password set
- MFA enrolled (required when org MFA policy is on)
- Role confirmed
- Location access confirmed
- Permissions reviewed
- PHI notice accepted
- Workflow intro completed

## Owner/admin view

Shows each active staff user with step completion badges and actions to mark progress.

## Product expectations

- Incomplete onboarding surfaces in admin notifications
- MFA gate can still block PHI access independently of checklist completion
- Checklist is operational status, not a substitute for HIPAA training attestation in production
